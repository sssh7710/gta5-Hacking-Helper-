# 저장소 작업 지침

이 파일은 저장소 전체에 적용됩니다.

## 기본 원칙

- 모든 설명, 작업 진행 상황, 결과 보고는 한국어로 작성합니다.
- 수정 전 관련 파일, 기존 구현, 설정, 문서를 먼저 확인하고 실제 내용과 오류 메시지를 근거로 판단합니다.
- 요청받지 않은 대규모 리팩터링, 파일 구조 변경, 실행 방식 변경은 하지 않습니다.
- 사용자의 명시적 허락 없이 새 라이브러리나 프레임워크를 추가하지 않습니다.
- 기존 `config.json` 필드, 저장 형식, 기본값, 사용자 설정과 프로그램 동작의 하위 호환성을 유지합니다.
- Windows 11, VS Code, Python 가상 환경을 우선 기준으로 작업합니다.
- 이 프로젝트는 사용자 PC에서 실행하는 일반 프로그램입니다. 서버 배포, 클라우드 운영, 웹 서비스, 데이터베이스 배포 자동화를 추가하지 않습니다.
- 게임 입력 자동화, 게임 프로세스·메모리 접근, DLL 삽입, 네트워크 조작처럼 기존 안전 범위를 벗어나는 기능은 요청 없이 추가하지 않습니다.

## 보안과 개인정보

- 비밀번호, API 키, 토큰, 쿠키, 세션, 개인 경로, 개인정보를 코드·문서·테스트 데이터·로그에 넣지 않습니다.
- `config.json`과 `diagnostics/`는 사용자별 로컬 데이터로 취급하고 Git에 포함하지 않습니다.
- 진단 화면에는 사용자 이름, 채팅 등 개인정보가 포함될 수 있으므로 예제나 이슈에 첨부하기 전에 반드시 확인하고 가립니다.

## 실행과 검증

- 일반 실행 방식은 `run.bat`이며, 준비 작업은 `setup.bat`을 사용합니다. 이를 임의로 바꾸지 않습니다.
- 별도 실행 파일 빌드 단계는 없습니다. 테스트는 가상 환경에서 다음 명령으로 실행합니다.

```powershell
.\.venv\Scripts\python.exe -B -m unittest discover -s tests -v
```

- GUI, 화면 캡처, 오버레이, 음성처럼 자동화하기 어려운 기능은 자동 테스트 결과와 수동 확인 결과를 구분합니다.
- 수정 후 변경 파일과 변경 이유, 실행한 명령, 테스트 결과, 남은 문제와 수동 확인 항목을 보고합니다.

## 베타 릴리스

- 기존에 공개한 베타 태그는 이동하거나 덮어쓰지 않고 새 베타 버전을 사용합니다.
- 베타 릴리스마다 직전 베타 태그 이후 수정된 Git 추적 파일을 경로 구조 그대로 ZIP으로 묶어 릴리스 Assets에 첨부합니다.
- 변경 파일 ZIP과 함께 SHA-256 체크섬 파일을 첨부합니다.
- `config.json`, `diagnostics/`, `.venv/`, 로그, 쿠키, 인증 정보와 기타 사용자별 파일은 릴리스 자산에 포함하지 않습니다.
- 실행 파일, 설치 프로그램 또는 자동 릴리스 기능은 사용자가 요청하지 않으면 추가하지 않습니다.
- 릴리스 완료 후 태그 대상 SHA, 첨부 자산 이름과 개수, 체크섬, 릴리스 URL, 최종 Git 상태를 확인합니다.
